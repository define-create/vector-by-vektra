Root Cause & Changes

Recharts' <Cursor> (the vertical line) is rendered as part of the <Tooltip> pipeline. It positions itself at the raw pointer X coordinate. The activeDot on <Line> positions itself at the snapped data-point X coordinate. With 10 points across a 400 px chart (~40 px per segment), a touch 20 px past a point leaves the cursor 20 px from the dot. On mobile this is consistently unreliable.

The fix removes the floating tooltip entirely and replaces it with a deterministic display driven by state.activeTooltipIndex — an integer index that Recharts internally uses to snap the activeDot. Both the info row and the dot now reference the same index, so they can never disagree.

REMOVE<Tooltip> from JSX and import — this also removes the cursor line
ADDuseState<number | null>(null) for activeIndex
ADDonMouseMove / onTouchMove → reads state.activeTooltipIndex, sets activeIndex
ADDonMouseLeave / onTouchEnd → resets activeIndex to null (shows latest point)
ADDInfo row above chart: date (left, muted) + rating (right, bold large) — updates via state
KEEPactiveDot, CartesianGrid, XAxis, YAxis, dot logic, yMin/yMax — all unchanged
